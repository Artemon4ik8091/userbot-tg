# **🛠 Полное руководство по созданию модулей для Юзербота**

Добро пожаловать в документацию для разработчиков\! Архитектура нашего юзербота построена на плагинах (модулях). Это означает, что **тебе никогда не нужно редактировать ядро (userbot.py)**, чтобы добавить новую функцию.  
Достаточно просто создать новый файл .py в папке modules/, и бот автоматически подхватит его при следующем запуске.

## **📑 Оглавление**

1. [Базовая структура модуля]
2. [Создание команд]
3. [Фоновые задачи (Background Tasks)]
4. [Работа с конфигурацией (Global\_config.json)]
5. [Полезные методы Telethon (Шпаргалка)]
6. [Правила и частые ошибки]

## **1\. Базовая структура модуля**

Каждый модуль должен взаимодействовать с ядром через registry.py. В начале файла всегда задавай метаданные модуля (его имя и описание), чтобы пользователям было понятно, за что он отвечает.  
**Минимальный шаблон модуля (modules/my\_module.py):**  
from registry import register\_cmd, set\_module\_meta

\# 1\. Задаем метаданные (Имя, Описание)  
set\_module\_meta("Мой Модуль", "Делает крутые штуки")

\# 2\. Пишем команды или фоновые задачи...

## **2\. Создание команд**

Команды вызываются, когда ты пишешь в чат .имя\_команды.  
Для создания команды используется декоратор @register\_cmd("имя", desc="описание").

### **Сигнатура функции команды**

Функция команды **обязательно** должна быть асинхронной (async def) и принимать ровно три аргумента:

1. client: Экземпляр TelegramClient (позволяет вызывать любые методы API).  
2. event: Объект события сообщения (содержит чат, текст, отправителя).  
3. args: Строка с текстом, который был написан *после* команды (без точки и самой команды).

### **Пример команды с аргументами:**

from registry import register\_cmd, set\_module\_meta

set\_module\_meta("Утилиты", "Базовые утилиты")

@register\_cmd("say", desc="Печатает текст от твоего имени")  
async def say\_cmd(client, event, args):  
    \# Если аргументов нет, ругаемся  
    if not args:  
        return await event.edit("❌ Укажи текст: \`.say Привет\`")  
      
    \# Меняем наше сообщение с командой на переданный текст  
    await event.edit(f"🗣 {args}")

*Если ты напишешь .say Привет мир, то args будет равно "Привет мир".*

## **3\. Фоновые задачи**

Фоновые задачи загружаются при старте бота и крутятся в бесконечном цикле на фоне. Идеально подходят для авто-статусов, чекеров, спаммеров или таймеров.  
Используйте декоратор @register\_bg(). Функция принимает только один аргумент: client.

### **Пример фоновой задачи:**

from registry import set\_module\_meta, register\_bg  
import asyncio

set\_module\_meta("Авто-Статус", "Меняет био каждую минуту")

@register\_bg()  
async def auto\_bio\_loop(client):  
    print("\[Авто-Статус\] Цикл запущен\!")  
      
    while True:  
        try:  
            \# Твоя логика здесь  
            pass  
              
        except Exception as e:  
            print(f"Ошибка в фоне: {e}")  
              
        \# ОБЯЗАТЕЛЬНО: спим, чтобы не повесить всего бота  
        await asyncio.sleep(60) \# Сон 60 секунд

**⚠️ ВАЖНО:** Никогда не используйте time.sleep(). Всегда используйте await asyncio.sleep(). Блокирующие функции остановят весь юзербот\!

## **4\. Работа с конфигурацией**

Бот оснащен встроенной системой конфигураций. Данные хранятся в файле Global\_config.json. Любой параметр можно изменить прямо из телеграма командой .cfg.  
Чтобы твой модуль работал с конфигом, импортируй: init\_config, get\_config, set\_config.

### **Шаг 1: Инициализация по умолчанию**

Вызови init\_config в начале файла, чтобы при первом запуске в JSON сразу прописались дефолтные настройки.  
from registry import register\_cmd, set\_module\_meta, init\_config, get\_config

set\_module\_meta("АнтиСпам", "Удаляет плохие слова")

\# Регистрируем дефолтные настройки (ключом выступает имя файла \_\_name\_\_)  
init\_config(\_\_name\_\_, {  
    "enabled": True,  
    "bad\_word": "редиска"  
})

### **Шаг 2: Чтение настроек внутри команды**

Используй get\_config(\_\_name\_\_, "имя\_ключа").  
@register\_cmd("check", desc="Проверить статус антиспама")  
async def check\_spam(client, event, args):  
    \# Получаем актуальные настройки  
    is\_on \= get\_config(\_\_name\_\_, "enabled")  
    word \= get\_config(\_\_name\_\_, "bad\_word")  
      
    status \= "ВКЛЮЧЕН ✅" if is\_on else "ВЫКЛЮЧЕН ❌"  
    await event.edit(f"Антиспам: {status}\\nЗапрещенное слово: {word}")

*Пользователь сможет поменять это слово командой: .cfg set modules.module\_antispam bad\_word дурак.*

## **5\. Полезные методы Telethon (Шпаргалка)**

Внутри функций (где есть client и event) тебе доступна вся мощь библиотеки Telethon. Вот самые частые операции:

### **Работа с текущим сообщением (event)**

* await event.edit("Текст") — изменить свое сообщение с командой.  
* await event.respond("Текст") — отправить *новое* сообщение в этот же чат.  
* await event.delete() — удалить сообщение с командой.

### **Получение информации о чате/отправителе**

* chat\_id \= event.chat\_id — ID текущего чата.  
* sender \= await event.get\_sender() — получить объект пользователя, с которым общаешься (в ЛС).  
* chat \= await event.get\_chat() — получить объект чата/группы.

### **Ответ (Reply) на чужое сообщение**

Часто команды применяются реплаем (например, .ban в ответ на сообщение хулигана).  
reply\_msg \= await event.get\_reply\_message()  
if reply\_msg:  
    print("Текст оригинального сообщения:", reply\_msg.text)  
    print("ID автора оригинального сообщения:", reply\_msg.sender\_id)  
else:  
    await event.edit("Эту команду нужно использовать ответом (реплаем) на сообщение\!")

### **Отправка медиафайлов**

\# Отправить картинку в текущий чат  
await client.send\_file(event.chat\_id, 'path/to/photo.jpg', caption="Смотри какая фотка\!")

### **Скачивание медиа (если команда вызвана реплаем на фото)**

reply \= await event.get\_reply\_message()  
if reply and reply.media:  
    \# Скачает файл в корень папки бота  
    path \= await client.download\_media(reply.media)  
    await event.edit(f"✅ Файл сохранен сюда: {path}")

## **6\. Правила и частые ошибки**

1. **Не используй time.sleep()**. Это заблокирует асинхронный цикл (Event Loop) и бот перестанет реагировать на другие команды. Используй await asyncio.sleep().  
2. **Перехват ошибок**. Оборачивай потенциально опасный код (парсинг сайтов, удаление чужих сообщений) в try...except, иначе ошибка вылетит в консоль, а сообщение с командой зависнет.  
3. **Не трогай event.out**. Ядро (userbot.py) уже фильтрует сообщения и передает в модули только *твои исходящие* сообщения. Тебе не нужно проверять, кто написал команду.  
4. **Уникальные имена команд**. Если два модуля зарегистрируют команду с одинаковым именем (например, @register\_cmd("info")), загрузится только последняя. Придумывай уникальные названия.  
5. **Импорты**. Декораторы и конфиг импортируй **только** из registry, а не из userbot.
